    nginx¶

    构建¶

    nginx:
    restart: always
    image: nginx:latest
    ports:
      - 80:80
      - 443:443
    volumes:
      - ./src/conf/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./src/conf_d:/etc/nginx/conf.d:ro
      - ./src/logs:/var/log/nginx
      - ./src/share:/usr/share

    配置¶

    nginx.conf

    user  nginx;
worker_processes  1;

error_log  /var/log/nginx/error.log warn;
pid        /var/run/nginx.pid;

events {
    worker_connections  1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;
    client_max_body_size 100M;

    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    access_log  /var/log/nginx/access.log  main;

    sendfile        on;#减少拷贝和模式切换
    tcp_nopush     on;#减少网络拥塞,会增加延迟

    keepalive_timeout  65;

    gzip  on;
    #只有超过1k的文件才压缩
    gzip_min_length 1k;
    #对文本文件压缩,不对图片压缩(意义不大,甚至有点亏)
    gzip_types text/plain application/javascript text/css application/font-woff;
    #低版本IE禁用Gzip压缩
    gzip_disable "MSIE [1-6]\."
    #每个内存页为4k,申请32倍
    gzip_buffers 32 4k;
    #压缩等级,1级压缩率已经很高,考虑到网速越来越快,1级压缩已经足够.
    gzip_comp_level 1;

    #静态资源缓存,对压缩后的资源进行缓存,大幅减少cpu占用
    proxy_temp_path /nginx-temp;
    #levels代表采用1:2也就是两级目录的形式保存缓存文件(静态资源css,js)，同时文件名进行了md5编码
    #keys_zone 定义缓存的名称及内存的使用，名称为babytun-cache,在内存中开始100m交换空间
    #inactive=7d 如果某个缓存文件超过7天没有被访问，则删除,如果7天内访问过，则延续7天
    #max_size=20g 代表设置文件夹最大不能超过20g,超过后会自动将访问频率（命中率）最低的缓存文件删除
    proxy_cache_path /nginx-cache levels=1:2  keys_zone=babytun-cache:100m  inactive=7d  max_size=20g;

    #服务器池配置,为微服务框架集成,灰度部署及负载均衡做准备
    #开发环境部署一个就够了
    upstream api{
       #最多失败1次 连接2秒就要切换其他服务器
       server api:80 weight=2 max_fails=1 fail_timeout=5s;
    }

    include /etc/nginx/conf.d/*.conf;
}

    使用心得¶

    平时使用nginx主要是为了实现端口复用,实验室开端口不容易,上次受到攻击后端口只剩一个了,故放弃控制,用nginx实现一个端口访问所有资源(按需配置server便可).两个端口则可以实现全内网控制.
