    SSH¶

    SSH升级¶

    # ssh升级
yum update openssh -y &&# 升级到yum能达到的最新版本
# yum install xinetd telnet-server -y # 安装telnet-server以及xinetd
# 配置telnet
# /etc/securetty文件末尾增加一些pts终端
# xvc0
# pts/0
# pts/1
# pts/2
# pts/3
# 启动telnet服务，并设置开机自动启动
# systemctl enable xinetd
# systemctl enable telnet.socket
# systemctl start telnet.socket
# systemctl start xinetd
# netstat -lntp|grep 23
# firewall-cmd --zone=public --add-port=23/tcp --permanent 
mkdir /data/tools -p &&
cd /data/tools &&
wget https://openbsd.hk/pub/OpenBSD/OpenSSH/portable/openssh-8.0p1.tar.gz &&
wget https://ftp.openssl.org/source/openssl-1.0.2s.tar.gz &&
tar zxf openssl-1.0.2s.tar.gz &&
mv /usr/bin/openssl /usr/bin/openssl_bak &&
mv /usr/include/openssl /usr/include/openssl_bak &&
cd /data/tools/openssl-1.0.2s/ &&
./config shared && make && make install &&
ln -s /usr/local/ssl/bin/openssl /usr/bin/openssl &&
ln -s /usr/local/ssl/include/openssl /usr/include/openssl &&
echo "/usr/local/ssl/lib" >> /etc/ld.so.conf &&
/sbin/ldconfig &&
cd /data/tools/ &&
tar xfz openssh-8.0p1.tar.gz &&
cd openssh-8.0p1 &&
chown -R root.root /data/tools/openssh-8.0p1 &&
rm -rf /etc/ssh/* &&
./configure --prefix=/usr/ --sysconfdir=/etc/ssh  --with-openssl-includes=/usr/local/ssl/include --with-ssl-dir=/usr/local/ssl   --with-zlib   --with-md5-passwords   --with-pam  && make && make install &&

# grep "^PermitRootLogin"  /etc/ssh/sshd_config yes
# grep  "UseDNS"  /etc/ssh/sshd_config no

cp -a contrib/redhat/sshd.init /etc/init.d/sshd &&
cp -a contrib/redhat/sshd.pam /etc/pam.d/sshd.pam &&
chmod +x /etc/init.d/sshd &&
chkconfig --add sshd &&
systemctl enable sshd &&
mv  /usr/lib/systemd/system/sshd.service  /data/ &&
chkconfig sshd on

# systemctl disable xinetd.service
# systemctl stop xinetd.service
# systemctl disable telnet.socket
# systemctl stop telnet.socket
# netstat -lntp

    注意: 改好后需做以下修改,保证外网访问.

    # /etc/ssh/sshd_config

PermitRootLogin yes
PubkeyAuthentication yes、
PasswordAuthentication yes。 

# /etc/selinux/config 
SELINUX=disabled

    防爆破¶

    SSHD不防爆破,需安装DenyHosts等服务,增加爆破成本.

    曾经靠DenyHosts让同学用刚学的卡里系统攻击赢了两瓶哇哈哈,而且似乎抵御了一次外网攻击吧.不明白明明只有我一台机子配了这denyhosts,整个实验室的机子的攻击次数都相同.按理说其它机子应该是每天几十万次才对.也许攻击者认为所有机子的配置都一样吧.

    DenyHosts功能一般,没记错的话fail2ban和fail2jail不仅防SSH爆破,还防DDOS攻击.具体可参照DDOS生存指南,后面有时间会进一步拓展.
